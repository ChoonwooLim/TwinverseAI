# Anthropic API

Claude 4.x 모델을 호출하는 Anthropic 공식 SDK / REST API 레퍼런스.

## SDK

- Python: `pip install anthropic` (공식)
- TypeScript: `npm i @anthropic-ai/sdk`
- REST 직접 호출 가능 (Bearer 인증)

## 기본 호출

```python
from anthropic import Anthropic
client = Anthropic()  # ANTHROPIC_API_KEY 환경변수
resp = client.messages.create(
    model="claude-opus-4-7",
    max_tokens=1024,
    messages=[{"role": "user", "content": "한국어로 답변"}],
)
print(resp.content[0].text)
```

## Prompt Caching

긴 시스템 프롬프트 · 반복 컨텍스트의 비용을 90% 이상 절감.

- **TTL**: 5 분 (마지막 사용 시점부터)
- **최소 단위**: 1024 토큰 (cache_control 블록 최소)
- **최대 4 개 블록** per request
- **캐시 히트 조건**: 모델 · 시스템 프롬프트 · cache_control 이 붙은 앞부분 완전 일치

```python
resp = client.messages.create(
    model="claude-opus-4-7",
    system=[
        {
            "type": "text",
            "text": LONG_SYSTEM_PROMPT,
            "cache_control": {"type": "ephemeral"},
        }
    ],
    messages=[{"role": "user", "content": USER_TURN}],
)
```

**비용 영향**:
- 캐시 write = input 단가의 1.25x (첫 호출만)
- 캐시 read = input 단가의 0.1x (이후 모든 히트)
- 대화 ≥ 3 턴에서 이미 손익분기 돌파

## Tool Use (Function Calling)

```python
tools = [{
    "name": "get_weather",
    "description": "도시의 현재 날씨를 조회",
    "input_schema": {
        "type": "object",
        "properties": {"city": {"type": "string"}},
        "required": ["city"],
    },
}]
resp = client.messages.create(
    model="claude-opus-4-7",
    max_tokens=1024,
    tools=tools,
    messages=[{"role": "user", "content": "서울 날씨 알려줘"}],
)
# resp.content 중 type=="tool_use" 인 블록 파싱
```

루프 패턴: tool_use → 결과 tool_result 로 다시 전달 → 최종 답변 수렴.

## Multimodal (Vision)

이미지 입력은 base64 인코딩해서 user 메시지에 포함:

```python
messages=[{
    "role": "user",
    "content": [
        {
            "type": "image",
            "source": {
                "type": "base64",
                "media_type": "image/png",
                "data": base64_str,
            },
        },
        {"type": "text", "text": "이 이미지를 설명"},
    ],
}]
```

- PNG / JPEG / GIF / WebP 지원
- 최대 5 MB per image
- 여러 이미지 동시 가능

로컬 대안: llava (Ollama) — [[Ollama-HTTP#multimodal]]

## Streaming

```python
with client.messages.stream(...) as stream:

```

## Streaming

기본값 `"stream": true` — NDJSON (줄당 JSON 객체) 반환. 각 청크에 `message.content` 누적 token.

```python
import requests, json
with requests.post("http://192.168.219.117:11434/api/chat",
                   json={"model": "qwen2.5:7b", "messages": [...]},
                   stream=True) as r:
    for line in r.iter_lines():
        if line:
            chunk = json.loads(line)
            print(chunk["message"]["content"], end="", flush=True)
```

## RPC 프레임 프로토콜 (native)

게이트웨이 ↔ 에이전트 통신 단위. WebSocket 프레임 상위에 native framing:

```
<4 bytes length> <1 byte type> <payload>
```

- type: `session.message`, `agents.list`, `agents.add`, `agents.update`, 등
- payload: JSON
- 프론트엔드는 payload 를 **unwrap** 해서 UI 에 표시 (2026-04-16 unwrap 누락으로 빈 화면 버그)

상세 파싱: `backend/routers/admin_openclaw_console.py`, `frontend/.../openclaw/ChatTab.jsx`.

|-------|------|
| session.read | 세션 조회 |

## Known Issues

### docker exec chown 이슈

`agents add` 를 컨테이너 내에서 실행하면 파일 소유자가 `root` 로 생성됨 → 에이전트 런타임 (node 사용자) 이 EACCES.
**해결**: `agents add` 후 즉시 `chown -R node:node /data/agents/<id>/` 수동 실행.
자동화는 gateway CLI 패치 대기.

### agents.update agentId 누락

- [ ] **Step 1: 스크립트 작성**

```powershell
#Requires -Version 7.0
# 주의: Windows PowerShell 5.1 은 UTF-8 no BOM `.ps1` 를 cp949 로 해석해 한글 깨짐.
# 반드시 pwsh 7+ 로 실행 (powershell.exe 아님).

<#
.SYNOPSIS
  AI_WIKI.md 포인터를 C:\WORK\ 하위 프로젝트들에 배포.

.PARAMETER Root
  스캔 시작 루트. 기본 C:\WORK.

.PARAMETER Template
  템플릿 파일 경로. 기본 스크립트 옆 AI_WIKI.template.md.

.PARAMETER Force
  기존 AI_WIKI.md 덮어쓰기.

.PARAMETER DryRun
  실제 파일 쓰지 않고 계획만 출력.

.EXAMPLE
  .\deploy-pointer.ps1
  .\deploy-pointer.ps1 -DryRun
  .\deploy-pointer.ps1 -Force -Root "C:\WORK"
#>

[CmdletBinding()]
param(
    [string]$Root = "C:\WORK",
    [string]$Template = (Join-Path $PSScriptRoot "AI_WIKI.template.md"),
    [switch]$Force,
    [switch]$DryRun
)

$ErrorActionPreference = "Stop"

# 스킵 목록 — 배포 대상 아님
$SkipList = @(
    'bin',           # 유틸리티
    'infra-docs',    # 자기참조 방지 (Registry 쪽)
    'llm-wiki',      # 자기자신 (Wiki repo)
    'UnrealEngine'   # 엔진 소스, 너무 큼
)

# 프로젝트 감지 마커
$ProjectMarkers = @(
    '.git',
    'package.json',
    'pyproject.toml',
    'requirements.txt',
    'Dockerfile',
    'CMakeLists.txt',
    'README.md'
)

function Test-IsProject {
    param([string]$Path)
    if (-not (Test-Path $Path -PathType Container)) { return $false }
    foreach ($marker in $ProjectMarkers) {
        if (Test-Path (Join-Path $Path $marker)) { return $true }
        # .uproject 는 이름이 가변이라 패턴 매칭
    }
    $uproject = Get-ChildItem -Path $Path -Filter "*.uproject" -File -ErrorAction SilentlyContinue | Select-Object -First 1
    if ($uproject) { return $true }
    return $false
}

function Deploy-Pointer {
    param(
        [string]$ProjectPath,
        [string]$TemplateContent,
        [switch]$Force,
        [switch]$DryRun
    )
    $projectName = Split-Path -Leaf $ProjectPath
    $targetPath = Join-Path $ProjectPath "AI_WIKI.md"

    if ((Test-Path $targetPath) -and (-not $Force)) {
        return [pscustomobject]@{
            Project = $projectName
            Action  = 'skipped-exists'
            Path    = $targetPath
        }
    }

    $content = $TemplateContent -replace '\{\{PROJECT_NAME\}\}', $projectName

    if ($DryRun) {
        return [pscustomobject]@{
            Project = $projectName
            Action  = 'dry-run'
            Path    = $targetPath
        }
    }

    # PowerShell 5.1 의 `Set-Content -Encoding UTF8` 은 BOM 을 붙이므로
    # .NET API 로 BOM 없는 UTF-8 작성 (git diff · 일부 markdown 파서 호환)
    [System.IO.File]::WriteAllText($targetPath, $content, [System.Text.UTF8Encoding]::new($false))
    return [pscustomobject]@{
        Project = $projectName
        Action  = if ($Force) { 'overwritten' } else { 'created' }
        Path    = $targetPath
    }
}

# ── 실행 ──

if (-not (Test-Path $Template)) {
    Write-Error "Template not found: $Template"
    exit 1
}
$templateContent = Get-Content -Path $Template -Raw -Encoding UTF8

if (-not (Test-Path $Root)) {
    Write-Error "Root not found: $Root"
    exit 1
}

$children = Get-ChildItem -Path $Root -Directory -ErrorAction SilentlyContinue

$results = @()
$skipped_list = @()
$skipped_not_project = @()

foreach ($child in $children) {
    if ($SkipList -contains $child.Name) {
        $skipped_list += $child.Name
        continue
    }
    if (-not (Test-IsProject $child.FullName)) {
        $skipped_not_project += $child.Name
        continue
    }
    $results += Deploy-Pointer -ProjectPath $child.FullName -TemplateContent $templateContent -Force:$Force -DryRun:$DryRun
}

# ── 리포트 ──

$created = @($results | Where-Object Action -in 'created', 'overwritten')
$skippedExists = @($results | Where-Object Action -eq 'skipped-exists')
$dryRunItems = @($results | Where-Object Action -eq 'dry-run')  # $dryRun 금지: $DryRun switch 와 대소문자 구분 없이 충돌

Write-Host ""
Write-Host "=== deploy-pointer 결과 ===" -ForegroundColor Cyan
Write-Host "Root: $Root"
Write-Host "Mode: $(if ($DryRun) { 'DRY RUN' } elseif ($Force) { 'FORCE (overwrite)' } else { 'normal' })"
Write-Host ""

if ($DryRun) {
    Write-Host "배포 예정 ($(($dryRunItems).Count) 건):" -ForegroundColor Yellow
    $dryRunItems | ForEach-Object { Write-Host "  - $($_.Project)" }
} else {
    Write-Host "배포 ($(($created).Count) 건):" -ForegroundColor Green
    $created | ForEach-Object { Write-Host "  - $($_.Project) [$($_.Action)]" }
}

Write-Host "스킵 (이미 존재, $(($skippedExists).Count) 건):" -ForegroundColor Gray
$skippedExists | ForEach-Object { Write-Host "  - $($_.Project)" }

Write-Host "스킵 (skip 목록, $($skipped_list.Count) 건): $($skipped_list -join ', ')" -ForegroundColor Gray
Write-Host "스킵 (비프로젝트, $($skipped_not_project.Count) 건): $($skipped_not_project -join ', ')" -ForegroundColor Gray

Write-Host ""
```
